---
title: "Quickstart"
description: "Install Kubescape and run your first Kubernetes security scan in under 5 minutes."
---

This guide walks you through installing Kubescape and running a full cluster scan. By the end you'll have a security posture overview for your cluster and know how to act on the results.

<Note>
  You need `kubectl` configured with access to a running Kubernetes cluster. Kubescape reads your current kubeconfig context.
</Note>

<Steps>
  <Step title="Install Kubescape">
    Run the one-line install script:

    ```bash
    curl -s https://raw.githubusercontent.com/kubescape/kubescape/master/install.sh | /bin/bash
    ```

    The script detects your OS and architecture, downloads the latest release, installs it to `~/.kubescape/`, and adds it to your PATH.

    Verify the installation:

    ```bash
    kubescape version
    ```

    <Tip>
      Looking for Homebrew, Windows PowerShell, or other package managers? See the full [installation guide](/installation).
    </Tip>
  </Step>

  <Step title="Run your first scan">
    Scan your current cluster with the default set of frameworks:

    ```bash
    kubescape scan
    ```

    Kubescape connects to your cluster via kubeconfig, evaluates all workloads and configurations against NSA-CISA, MITRE ATT&CK, and CIS Benchmark controls, and prints a summary to your terminal.
  </Step>

  <Step title="Understand the output">
    The scan output is divided into sections:

    **Control plane** shows the security posture of your Kubernetes API server and core cluster configuration. A checkmark (вң…) means the control passed; a cross (вқҢ) means it failed. Each failed control links to documentation with remediation steps.

    **Access control** lists RBAC-related controls and the number of resources that failed each one. The `Resources` column tells you how many objects are affected. Use the `View Details` command to drill in:

    ```bash
    kubescape scan control C-0035 -v
    ```

    **Secrets** highlights workloads or configurations that may be exposing credentials insecurely.

    **Network** reports workloads without network policies, which are a key isolation control.

    **Workload** covers pod-level security settings such as privileged containers, host PID/IPC access, and whether containers run as root.

    **Highest-stake workloads** identifies the workloads that would have the greatest blast radius if compromised. Scan them individually for a detailed vulnerability report:

    ```bash
    kubescape scan workload DaemonSet/gadget --namespace gadget
    ```

    **Compliance score** shows the percentage of controls that passed for each framework. A higher score means better compliance. Run a framework-specific scan to get the full report:

    ```bash
    kubescape scan framework nsa
    kubescape scan framework mitre
    ```
  </Step>
</Steps>
